# 보안과 개인정보

> English original: [../SECURITY.md](../SECURITY.md)

## 목적

사용자 입력을 다루고 로컬 Bluetooth 장치로 자신을 표시하는 앱의 신뢰 모델을 문서화합니다.

## 현재 상태

PhonePad v1.0에는 계정, 클라우드 동기화, 광고, 분석, 인터넷 권한이 없습니다.

## 현재 규칙

- 사용자 입력은 페어링된 호스트로 가는 Bluetooth HID 리포트 외에는 휴대폰을 벗어나면 안 됩니다.
- 텔레메트리를 수집하지 않습니다.
- Crash reporting은 opt-in, 익명, v1.0 이후 문서화 조건을 만족하지 않으면 추가하지 않습니다.
- 로컬 로그는 사용자가 복사할 수 있게 유지하고 자동 업로드하지 않습니다.
- 소스는 공개하고 빌드는 재현 가능하게 유지합니다.

## 데이터 처리

| 데이터 | 저장소 | 참고 |
|---|---|---|
| 페어링된 호스트 표시 이름/주소 | Local SQLite | 재연결과 호스트 목록에 필요. |
| OS 프리셋 | Local SQLite | 단축키 매핑에 필요. |
| 제스처 매핑 | Local SQLite | 사용자 설정. |
| 설정 | Local SQLite | 사용자 설정. |
| 포인터 이동 기록 | 저장하지 않음 | HID 리포트로만 처리. |
| 진단 로그 | 로컬 전용 | 사용자가 수동으로 복사/공유 가능. |

## 권한 자세

- v1.0에서 `INTERNET`은 금지합니다.
- Bluetooth 권한은 사용자 대면 한국어로 설명해야 합니다.
- Notification 권한은 foreground connection status로 정당화해야 합니다.
- `BLUETOOTH_SCAN`이 필요한 경우 이유를 문서화하고 Android가 허용하는 범위에서 위치 관련 표현을 피합니다.

## 오픈소스 신뢰 신호

- 공개 코드.
- 네트워크 권한 없음.
- 추적 SDK 없음.
- 재현 가능한 GitHub Actions 릴리스 빌드.
- 테스트 및 미지원 기기를 나열한 README 지원 매트릭스.

## 위협과 완화

| 위험 | 완화 |
|---|---|
| 과도한 권한이 신뢰를 낮춤. | Manifest를 최소화하고 각 권한을 문서화. |
| 마우스 버튼 고착이 호스트를 방해함. | 안전 해제 훅과 반복 Drag Mode 테스트. |
| 호스트 스푸핑 또는 잘못된 대상 선택. | 명확한 활성 호스트 UI와 전환 전 해제. |
| 로그가 개인 데이터를 유출함. | 원시 입력 스트림이나 입력된 콘텐츠를 로깅하지 않음. |
| 공급망 SDK 위험. | 광고, 분석, 불필요한 서드파티 SDK 회피. |
